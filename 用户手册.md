# Wego 用户手册

## 这个用户手册还没有完全完成，目前供开发人员参考

## 1. 手册目的
本文档用于指导用户在本地部署并使用 Wego 校园社区论坛系统，覆盖以下内容：
- 环境准备
- 系统部署
- 功能使用
- AI 服务接入
- 常见问题排查

## 2. 系统简介
Wego 是一个前后端分离的校园社区论坛项目，主要能力包括：
- 密码注册登录
- 文章发布、编辑、删除、搜索、点赞、评论
- 个人中心、关注/粉丝、消息通知、成就值排行
- 私信（功能未完全完成）

技术栈：
- 后端：Spring Boot + MyBatis + Redis + Quartz + WebSocket + ELK
- 前端：Vue + ElementUI + Axios
- AI 扩展服务：FastAPI（独立进程）

## 3. 部署拓扑与组件关系
- 前端（web）提供页面与交互
- Java 后端提供业务 API
- MySQL 存储业务数据
- Redis 用于缓存、验证码、异步事件等
- ELK 用于搜索相关能力（Elasticsearch + Logstash + Kibana） （可选）
- AI Service 为后端提供 7 项 AI 接口（可选）

## 4. 环境要求
建议在 Windows 环境部署（项目已提供 Windows 相关脚本）。

基础软件：
1. JDK 8+（建议与项目当前 Maven 配置一致）
2. Maven 3.6+
3. MySQL 8.0（MySQL 5.x 需调整 JDBC 配置）
4. Redis（Windows 版可用）
5. Node.js（原项目推荐 v15.14.0）
6. ELK 同版本套件（Elasticsearch、Logstash、Kibana）（可选）
7. Python 3.9+（用于 AI Service，可选）

## 5. 部署步骤

### 5.1 获取代码
在项目根目录执行后续步骤，目录中应包含：
- src（Java 后端）
- web（Vue 前端）
- wego.sql（数据库初始化脚本）
- ai-service（AI 独立服务）

### 5.2 初始化数据库
1. 在 MySQL 中创建数据库：wego
2. 导入项目根目录下的 wego.sql
3. 确认基础表已创建成功

### 5.3 配置后端
1. 打开 src/main/resources/application.properties
2. 修改数据库连接、用户名、密码
3. 修改 Redis 连接信息
4. 如果启用第三方能力，请配置以下密钥（可选）：
   - 七牛云对象存储
   - Elasticsearch 连接

注意：
- 若使用 ELK 同步 MySQL，请在相关 JDBC 配置中保留 tinyInt1isBit=false，避免 tinyint(1) 被转换为 boolean 导致类型错误。

### 5.4 启动依赖服务
按顺序启动：
1. MySQL
2. Redis
3. Elasticsearch / Logstash / Kibana（如需搜索能力）

### 5.5 启动 Java 后端
在项目根目录执行：

```bash
mvn clean spring-boot:run
```
或运行 src/main/java/com/wego/WegoApplication.java 中的 main 方法。

启动后检查日志，确认应用正常监听端口（以 application.properties 实际配置为准，常见为 8080）。

### 5.6 启动前端
在 web 目录执行：

```bash
npm install
npm run dev
```

启动后浏览器访问前端地址（以控制台输出为准，常见为 8081）。

### 5.7 可选：启动 AI Service
在 ai-service 目录执行：

```bash
python -m venv .venv
.venv\Scripts\activate
pip install -r requirements.txt
```

复制并配置环境变量：

```bash
copy .env.example .env
```

编辑 .env，至少配置 MODEL_API_KEY 等参数，然后启动：

```bash
uvicorn app.main:app --reload --port 8000
```

文档地址：
- http://localhost:8000/docs

## 6. 联调与验证

### 6.1 基础功能自检
1. 打开前端首页
2. 完成注册/登录（手机号验证码或密码）
3. 发布一篇文章
4. 在详情页执行点赞、评论
5. 在个人中心查看关注/粉丝、我的信息

### 6.2 搜索能力验证
1. 确认 ELK 已启动并完成数据同步
2. 在前端搜索文章关键词
3. 检查结果返回与高亮显示

### 6.3 AI 服务连通性验证（可选）
1. 访问 http://localhost:8000/health、/ready、/version
2. 测试接口，例如 POST /api/v1/summary
3. 请求头需携带：
   - X-Timestamp（毫秒级 Unix 时间戳）
   - X-Service-Key（HMAC-SHA256(timestamp + ":" + SERVICE_KEY, SERVICE_KEY)）

## 7. 日常使用指南

### 7.1 普通用户
- 注册并登录账号
- 浏览、搜索、点赞、评论文章
- 发布或编辑自己的文章
- 进入个人中心维护资料
- 管理关注/粉丝关系

### 7.2 内容发布建议
- 标题尽量包含主题关键词，便于搜索
- 正文分段清晰，避免一次性大段文本
- 上传图片前确认后端上传参数名一致（file）

### 7.3 通知与私信
- 点赞/评论/关注触发异步通知
- 私信功能可体验基础能力，但当前版本尚未完全完工

## 8. 常见问题与处理

1. 前端启动失败（npm install 报错）
- 检查 Node.js 版本，建议先使用 v15.14.0
- 删除 node_modules 后重新安装

2. 登录验证码不可用
- 检查 Redis 是否启动
- 检查短信服务配置是否正确

3. 搜索报类型转换错误（Boolean -> int）
- 检查 Logstash JDBC 配置是否包含 tinyInt1isBit=false

4. 上传图片空指针
- 检查前端上传字段名与后端接收参数名是否都为 file

5. 前后端无法联通
- 检查后端地址是否配置正确
- 如使用花生壳，检查映射是否生效
- 仅本地联调时可将线上地址替换为本地地址

## 9. 生产部署建议
1. 使用 Nginx 统一反向代理前后端
2. 将 MySQL、Redis、ES 与应用分机部署
3. 启用 HTTPS 与密钥集中管理
4. 为 AI Service 增加限流与重试机制
5. 配置应用日志轮转与监控告警
