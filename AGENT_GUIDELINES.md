# Agent 协同开发与代码规范指南

本文档用于记录和规范 AI Agent 在操作本项目代码时需要注意的强制性规则与易错点。Agent 在执行复杂重构或生成文件操作前，请务必参考本文档进行校验。

## 1. 文件编码规范 (关键)

**规则**：所有项目文件（尤其是前端项目库下的 `.vue`、`.js`、`.json` 文件，以及后端的 `.java`、配置文件等）**必须严格使用 UTF-8 编码**，且不得包含 BOM (Byte Order Mark)。

**背景与踩坑记录**：
- 在 2026 年 4 月的一次重构中，因底层 Windows PowerShell 默认输出机制的问题，导致 `src/views/login.vue` 文件被静默保存为了 `UTF-16LE` 编码格式。
- UTF-16 包含大量的 `\u0000` (NUL) 空字节，导致前端 `vue-loader` 无法正常解析 `<template>` 和 `<script>` 标签块，最终引发了 `[Vue warn]: Failed to mount component: template or render function not defined.` 错误，导致页面白屏组件失效。

**Agent 操作要求**：
1. **优先使用原生 API**：Agent 在新建文件、写入代码或修改代码时，请始终使用平台提供的代码读写工具（如 `create_file`、`replace_string_in_file`），这些接口能安全地保证 UTF-8 编码。
2. **禁用终端重定向覆写**：**坚决禁止**使用 PowerShell 命令（如 `echo "..." > file.vue`、`Out-File` 等重定向符号）去直接覆写或生成代码文件，除非显式且安全地指定了无 BOM 的 UTF-8 编码参数，以此杜绝次生隐患。

## 2. 接口修改与规范记录

**规则**：Agent 在工作时必须记录修改的接口，并记录接口规范。

**Agent 操作要求**：
1. **记录接口变更**：在修改、新增或废弃任何 API 接口（包括请求路径、参数、返回值结构等）时，必须明确记录这些改动内容。
2. **维护接口规范**：确保同步更新代码注释或相应的接口文档，保持接口定义与规范说明的一致性，以确保对接和后续维护的准确无误。

## 3. 长任务防范机制与状态同步

在执行复杂的全栈开发或重构等长周期任务时，Agent 极易受限于上下文长度和局部视野。必须严格遵守以下防范机制：

**Agent 操作要求**：
1. **阶段性总结与进度备忘**：在复杂任务中，强制要求进行子任务拆解。完成阶段性任务时，应将核心进展、已做出的技术决策、确定的规范等信息作梳理记录，防止随上下文增长而遗忘。
2. **全局状态与引用同步 (关键)**：当修改被复用的类（如 Entity 等）、全局组件或处理前后端接口对接时，**严禁只修改工作中的单点文件**。必须使用全局搜索工具（如 `grep_search`）查找所有引用处并同步修改，坚决避免前后端脱节及引用漏改。
3. **克制目标发散**：坚守核心任务目标。在主线任务未完成前，对沿途发现的无关代码异味 (Code Smell) 或轻微警告仅作留意记录，禁止钻牛角尖进行发散修改。
4. **规避试错死循环**：在处理特定的报错时如陷入僵局（基于同一套排查逻辑二次尝试依然报错），必须强制跳出死循环，转换思路或寻求全局替代方案，避免无限重复。
5. **控制终端日志输出**：在执行环境编译或依赖安装等高输出终端命令时，注意预防超长报错堆栈和进度刷屏日志导致上下文瞬间溢出并引发幻觉。
